\section{Methodology}

We benchmark the Forward and Viterbi dynamic programming algorithms across multiple
backends in the Machine Boss WFST toolkit:

\begin{itemize}
  \item \textbf{C++ native}: the \texttt{boss} CLI (\texttt{-L} for Forward, \texttt{-V} for Viterbi)
  \item \textbf{JAX 1D scan}: Python/JAX using \texttt{jax.lax.scan} (sequential)
  \item \textbf{JAX 1D parallel}: Python/JAX using \texttt{jax.lax.associative\_scan} (parallel prefix)
  \item \textbf{JAX 2D scan}: Python/JAX 2D with sequential outer scan
  \item \textbf{JAX 2D wavefront}: Python/JAX 2D with anti-diagonal wavefront parallelism
  \item \textbf{JAX GPU}: JAX on GPU (skipped if unavailable)
  \item \textbf{JS CPU}: JavaScript CPU fallback via Node.js
\end{itemize}

\subsection{1D problems (generators)}

Random generator machines (output-only) with DNA alphabet (A, C, G, T) are tested across:
\begin{itemize}
  \item Sequence length $L \in \{100, 500, 2000, 10000\}$
  \item Machine states $S \in \{4, 16, 64\}$
\end{itemize}

\subsection{2D problems (transducers)}

Random transducer machines with DNA alphabet are tested across:
\begin{itemize}
  \item Input length $L_{\mathrm{in}} \in \{10, 50, 200\}$
  \item Output length $L_{\mathrm{out}} \in \{10, 50, 200\}$
  \item Machine states $S \in \{4, 16, 64\}$
\end{itemize}

Each configuration is timed over multiple repetitions after a warmup call
(which also triggers JIT compilation for JAX backends).
We report mean $\pm$ standard deviation in seconds.
Configurations that exceed a timeout threshold are recorded with their probe timing
and larger sizes are skipped.
